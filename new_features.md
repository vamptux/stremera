## ⚡ Performance & Hardware Efficiency Optimizations
*   **Wire Up `@tanstack/react-virtual` (Already Installed!):** The dependency is in `package.json` but not used anywhere. Apply virtualization to: (a) search results grid (can reach 2,000 items), (b) library/watchlist grid, (c) watch history list, (d) episode lists on details page, (e) stream selector results. This will eliminate the frame drops users experience on large collections without adding any new dependencies.
*   **Offload Accent Color Extraction to Web Worker:** The `details.tsx` accent color extraction runs `canvas.drawImage()` + pixel-by-pixel sampling synchronously on the main thread every time a details page loads. Move this computation to a Web Worker (or use `fast-average-color`'s built-in async mode) to prevent the 50–200ms UI jank on lower-end hardware.
*   **History Index by Media ID for O(1) Lookups:** `get_watch_history_for_id` iterates the entire history index to filter by one media ID — O(n) per call. Add a secondary index map (`media_id → Vec<history_key>`) maintained alongside the primary index in `history.json`. This makes per-show history lookups instant even with 10,000+ entries.
*   **Run Store Migration Once at Startup Only:** `load_or_migrate_*` functions in `commands.rs` run on every single command invocation, checking for legacy data format even after migration is complete. Add a one-time migration flag (`_migrated: true`) to each store so subsequent commands skip the migration codepath entirely.
*   **Player Re-Render Boundaries with `React.memo`:** With ~35 `useState` variables, any state change in `player.tsx` re-renders the entire 3,240-line tree including the bottom controls bar, episode panel, and stream selector. Extract `<PlayerControls>`, `<EpisodePanel>`, `<StreamSelectorPanel>` as `React.memo` components with stable callback refs to create render isolation boundaries.
*   **Lazy-Load List Items in `get_lists`:** The `get_lists` Rust command loads every item in every list on every call, including full `MediaItem` JSON. For users with 20 lists × 50 items each, this transfers ~1,000 media items per invocation. Add pagination or lazy hydration — return list metadata first, then hydrate items on demand when a list is expanded.
*   **Debounce Media Card Hover Fetches:** Each `MediaCard` fires `getMediaDetails()` after only 80ms hover delay. Rapid mouse movement across a row of 20 cards creates 20 concurrent API calls. Increase the hover-to-fetch delay to ~400ms and cancel pending fetches on mouse leave. The Torrentio 180s cache mitigates repeat calls but doesn't prevent the initial burst.
*   **Optimize Rust Release Profile for Size + Speed:** The current release profile uses `lto = "thin"` and `codegen-units = 8`. For a shipping binary, switch to `lto = "fat"` and `codegen-units = 1` for maximum optimization. Add `opt-level = 3` explicitly. This can reduce binary size by 10–20% and improve hot-path performance (stream resolution, torrent matching regexes).
*   **Precompile Torrentio Regexes at Build Time:** The Torrentio provider uses `LazyLock` for regex compilation — good — but the ~15 regex patterns for batch detection, episode matching, quality scoring, and emoji parsing are complex. Validate they compile only once and add `#[cfg(test)]` unit tests to catch regex regressions. Consider using `regex::RegexSet` for the quality scoring pass where multiple patterns are tested against the same input.

Add a configurable auto-retry policy (default: 3 retries with 30s → 60s → 120s backoff). For HTTP 416 (range not satisfiable), automatically reset the download offset. *   **Unified User-Agent Constant:** Each provider (`cinemeta.rs`, `kitsu.rs`, `netflix.rs`, `torrentio.rs`, `realdebrid.rs`) and the downloader independently define a Chrome user-agent string. Add tests for: (a) `resolve_stream_inner` with mock HTTP responses, (b) `get_streams` parsing with sample Torrentio JSON, (c) history save/load round-trip, (d) library CRUD operations, (e) download lifecycle state transitions. *   **`#[allow(clippy::too_many_arguments)]` Cleanup:** Multiple functions suppress the too-many-arguments lint. Introduce config/request structs: `StreamQuery { media_type, id, season, episode, absolute_episode }`, `ResolveOptions { magnet, info_hash, file_idx, url }` — improving readability and enabling future field additions without breaking signatures.
*   **Store File Locking for Concurrent Access Safety:** Multiple Tauri commands can race on the same `.json` store file (e.g., two rapid `save_watch_progress` calls). While `tauri-plugin-store` has internal locking, the read-modify-write pattern across multiple store calls within a single command is not atomic. ## 🖥️ Platform & System Polish
*   **Proper Windows Title Bar with Custom Controls:** The app uses `transparent: true` in `tauri.conf.json` but doesn't implement custom window controls (minimize/maximize/close). Add a sleek custom title bar component with Tauri's `appWindow.minimize()` / `.toggleMaximize()` / `.close()` that blends with the app's theme — replacing the default OS chrome or the lack thereof.
*   **Multi-Monitor Awareness (Remember Window Position):** Persist the window's last position, size, and monitor index to `settings.json` on close. Restore to the same position on relaunch. Handle the case where the saved monitor is no longer connected (fall back to primary display center). Currently the window always opens at the OS default position.
*   **Startup Performance Profiling & Splash Screen:** The app runs `migrate_legacy_app_data_stores()`, loads 5 JSON stores, initializes 6 provider HTTP clients, and copies MPV DLLs at startup. Add a lightweight splash screen (300ms minimum display) that masks the initialization time, and log timing spans via `tracing` so startup regressions are detectable.
*   **Graceful Shutdown with State Flush:** On `window-close-requested`, ensure all pending operations complete: (a) flush final `save_watch_progress` bypassing the 15s debounce, (b) persist any in-memory download state changes, (c) abort active download tasks cleanly (not hard-kill), (d) save window geometry. Currently the app relies on OS process termination which can lose the last ~15s of watch progress.
*   **Portable Mode (Config Next to Binary):** Detect a `portable.txt` sentinel file next to the executable. When present, redirect all data storage (`settings.json`, `history.json`, etc.) to a `data/` folder next to the binary instead of `%APPDATA%`. Enables running Streamy from a USB drive with all settings intact.
*   **Single Instance Lock:** Prevent multiple Streamy windows from running simultaneously, which would cause store file corruption. Use `tauri-plugin-single-instance` to focus the existing window when a second launch is attempted, optionally accepting a deep-link URL argument.

Some users on older GPUs or VMs need to force software decoding. Currently it defaults to whatever libmpv decides — giving users control prevents "black screen on play" issues from GPU driver bugs.

## 🔐 Security Improvements
*   **Download URL Validation & Sanitization:** `start_download` accepts any URL string from the frontend with no validation. Add scheme checks (only `http://` and `https://`), host validation (block `localhost`, `127.0.0.1`, `0.0.0.0`, private IP ranges), and path traversal checks. Prevents a compromised frontend or XSS from triggering downloads of arbitrary files.
*   **`open_folder` Path Traversal Hardening:** The `open_folder` command accepts arbitrary paths from the frontend with only an empty-string check. Validate that the path is within the app's download directory or app data directory before calling `explorer /select`. Prevents the command from being used to probe the filesystem.
*   **File Path Sandboxing for Import/Export:** `export_app_data_to_file` and `import_app_data_from_file` accept arbitrary filesystem paths from the frontend. Restrict these to the OS-standard Documents or Downloads directories, or use `tauri-plugin-dialog`'s file picker exclusively (which is already imported but the commands accept raw paths as well).
*   **CSP Tightening for Production:** The current CSP allows `connect-src 'self' https: ws: wss:` which permits connections to any HTTPS host. Restrict to known provider domains: `*.strem.io`, `*.strem.fun`, `*.real-debrid.com`, `*.kitsu.io`, `*.kitsu.app`, `api.aniskip.com`, `api.introdb.app`, `*.baby-beamup.club`. This prevents data exfiltration if an XSS vulnerability is ever introduced.
